## Python 数据分析标准流程备忘录

> 以下内容适用于本人的 Mac OS。部分代码不适合直接作用在 Windows 或者其他系统。



### 1. 项目与环境管理

这是确保项目可维护性的第一步。

- **创建项目文件夹**：为项目创建一个独立的文件夹，和目录。或者使用 VS code 直接打开某个文件夹，VS Code会自动将该文件夹识别为一个 Workspace。

> 在 R 和 Matlab 上，有项目（Project）这一概念，在 VS Code 和 Positron 中，有一个类似的概念，Workspace。R 和 Matlab 上需要在某一文件夹中，用一个配置文件 (`.Rproj`, `.prj`) 来表示该项目。但是VSCode中对于 Workspace 的定义就是一个文件夹，不太会专门为了这个文件夹追加一个配置文件来表示这是一个workspace。除非是 多根工作区，会有一个 `.code-workspace` 文件。


    
- **建立虚拟环境**：为每一个项目单独创建一个虚拟环境，来保证该项目只调用该虚拟环境中的库和包，从而保证该项目和其他项目之间的库和包版本不会互相冲突。

    
    - 方法1: 使用终端和 `venv`：在终端运行 `python -m venv venv`，即可自动创建一个虚拟环境。但是该环境此时并未激活，Mac OS 使用代码 `source venv/bin/activate` 来激活该虚拟环境。
        
    - 方法2: 在 vscode 中，而不是终端，可以直接使用 `Shift + Command + P` 调出命令面板（Command Palette），然后搜索并选择，`Python: Create Environment...`，然后按提示操作即可。

> 也可以使用终端和 `conda` （如果你使用 Anaconda）来创建虚拟环境。本系统使用 Homebrew，因此不使用 conda 来管理虚拟环境。

### 2. 数据导入与清洗

这是数据处理的基石，**`pandas`** 是核心工具。

- **导入数据**：使用 `pandas` 的 `read_csv()`、`read_excel()` 等函数将数据加载到 **DataFrame** 中。DataFrame 是 `pandas` 的核心数据结构，非常类似于 R 的数据框。
    
- **探索性数据分析 (EDA)**：使用 `DataFrame.info()`、`DataFrame.describe()`、`DataFrame.isnull().sum()` 等方法快速了解数据的结构、类型、缺失值和统计概况。
    
- **数据清洗与转换**：利用 `pandas` 提供的强大方法进行数据处理。这包括：
    
    - **筛选**：使用布尔索引（boolean indexing）来选择行。
        
    - **切片**：使用 `loc[]` 和 `iloc[]` 来选择行和列。
        
    - **缺失值处理**：使用 `fillna()` 或 `dropna()` 方法。
        
    - **数据类型转换**：使用 `astype()` 方法。
        
    - **数据合并**：使用 `merge()` 或 `concat()` 函数。
        

> 这一步骤细节暂时未考察，但是 pandas 确实是一个非常知名的 package。




### 3. 数据可视化

通过图形来洞察数据中的模式和趋势。

- **基础绘图**：通常会用到 `Matplotlib`，这是 Python 最基础的绘图库。
    
- **高级绘图**：`Seaborn` 和 `Plotly` 是更常用的选择。
    
    - **`Seaborn`**：基于 `Matplotlib`，提供更美观、更高级的统计图表，如热力图、小提琴图等。
        
    - **`Plotly`**：可以创建交互式图表，非常适合在网页上展示。
        


> 这一步骤细节暂时未考察，但是 Matplotlib 是一个知名的 package。




### 4. 统计建模与机器学习

Python 在这方面生态极其丰富，这是它和 R 最大的区别之一。

- **统计分析**：`statsmodels` 是 Python 中进行计量经济学和传统统计分析的首选，它提供了丰富的统计模型，如线性回归、时间序列分析等。
    
- **机器学习**：`scikit-learn` 是这个领域最核心的库。它提供了几乎所有常见的机器学习算法，从回归、分类到聚类，以及模型评估和交叉验证工具。
    

> 这一步骤暂时未考察。



### 5. 结果交流与分享

- **生成报告**：使用 **Jupyter Notebook** 或 **JupyterLab** 是最常见的做法。它们能将代码、输出、图表和文字说明集成到一份文档中，非常便于分享和复现。
    
- **项目归档**：使用 `pip freeze > requirements.txt` 命令将你项目中所有安装的库和版本信息保存到一个文件中。当其他人需要运行你的项目时，他们只需运行 `pip install -r requirements.txt`，就能精确地重建你的开发环境。
    

> pip freeze 这一命令在venv使用过程中有所了解。





这个流程和 R 的流程在思路上高度一致，都遵循**“数据获取 -> 清洗 -> 可视化 -> 分析 -> 分享”**的模式。主要的区别在于所使用的具体工具和库。

如果你同时使用 R 和 Python，你可能会发现这两种语言在处理不同任务时各有优势。比如，R 在统计学方面有更深厚的积淀，而 Python 在机器学习和软件工程方面更具优势。
